什么是 MCP？

MCP（Model Context Protocol，模型上下文协议）是 Anthropic 于 2024 年 11 月发布的开放标准，采用 JSON-RPC 2.0 传输格式，致力于在 AI 模型与外部工具 / 数据源之间建立统一、双向、安全的接口。MCP 解决了大模型在长对话、多轮交互中因上下文窗口有限导致的信息丢失，同时降低了 Agent 调用外部工具的开发门槛。

得益于 MCP，开发者只需“写一次连接器”，模型便可在安全沙盒内调用浏览器、数据库、Git 仓库、地图服务等海量资源。

核心组成

MCP 主机
(Hosts)

发起请求的 AI 应用，如聊天机器人、IDE、桌面助手等，为模型提供运行环境与用户交互界面。

MCP 客户端
(Clients)

嵌入主机内部，负责与多个 MCP 服务器建立 1:1 安全连接，充当“总机”与“路由器”。

MCP 服务器
(Servers)

每个服务器专攻一类任务：读写文件、查询数据库、操作 Git、调用地图 API……为模型提供工具、资源与提示。

主要特点

动态上下文注入

在推理过程中按需注入工具描述、文档与用户状态，无需硬编码 Prompt。

本地 + 远程数据

既能离线检索本地文件，也能安全调用云端 API；敏感数据可完全留在用户设备。

多语言 SDK

官方提供 Python、TypeScript、Java 等 SDK 与 CLI，轻松实现、调试与部署。

安全优先

内置用户同意、数据隐私、工具沙箱、LLM 采样控制等安全原则；MCP Inspector 工具助力调试。

开放生态

已涌现百余个社区 Server：从 Google Drive 到 PostgreSQL，应有尽有。

协议友好

借鉴 LSP 思路，基于 JSON-RPC，易于扩展，可与任意支持 MCP 的模型对接。

MCP 的优势

  • 开放标准：避免不同厂商 Function Call API 的差异，实现一次接入、处处可用。
  • 减少重复开发：复用现有 Server 与 SDK，专注业务逻辑。
  • 模型无关：任意支持 MCP 的模型（Claude、GPT、Gemini…）均可切换。
  • 数据留边界：本地 Server 让敏感数据“不出门”，最大化隐私与合规。

应用场景

阿里云百炼平台

提供全生命周期 MCP 服务：5 分钟创建智能体，自动完成从资源管理到运维部署。

高德地图 MCP Server

整合 12 项核心地图能力，助力路线规划、POI 查询、气象可视化等企业级场景。

IDE 与 DevOps

Replit、Zed、Sourcegraph 等开发平台通过 MCP 让 AI 助手实时读取项目上下文、创建 PR、自动审查代码。
